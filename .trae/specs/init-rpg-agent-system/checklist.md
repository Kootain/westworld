# Checklist

本清单用于在开发和验收过程中逐一核对需求文档中定义的核心指标与规范。

## 阶段一：文档与架构设计验收
- [x] 需求文档 (`spec.md`) 已清晰描述三层架构解耦、三级记忆、以及认知链条。
- [x] 代码规范已明确声明 Clean Code、单一职责、依赖倒置原则。
- [x] 目录结构划分 (`io`, `box`, `memory`, `agent`, `graph`, `core`) 已在规范中明确。

## 阶段二：代码实现质量审查 (Code Quality)
- [ ] 所有的 I/O 都必须通过 Adapter 接口调用，全局业务代码中不得出现 `print()` 和 `input()`。
- [ ] 所有的函数、类必须包含完整的 Type Hints 和 Docstrings。
- [ ] 外部数据（特别是 LLM 的返回）必须通过 Pydantic 进行校验。
- [ ] 关键逻辑节点必须包含标准 `logging` 日志输出，无乱码和调试用冗余输出。

## 阶段三：核心功能验收 (Functional Tests)
- [ ] **拦截机制测试**：将角色状态设为 `SLEEPING`，发送任意文本，断言能立刻收到 Box 生成的系统回复，且 LLM 不被触发。
- [ ] **记忆衰减测试**：连续输入大量对话，断言 STM 能够根据设定的 Token 阈值及时间衰减公式，自动逐出分数最低的记录。
- [ ] **记忆固化测试**：断言异步 Consolidation 任务能够成功提取“核心事实”，并写入 LTM 存储。
- [ ] **隐私屏蔽测试**：断言 Agent 生成的带有 `SoT` (内心独白) 的 JSON，在经过 Box 处理后，控制台展示的信息中绝对不包含 `SoT` 字段。
- [ ] **LangGraph 运转**：断言 Perceive -> Recall -> SoT -> Act 的工作流能稳定流转。
- [ ] **主动外显测试**：断言后台 Tick 系统能够在长时间无输入时，自动产生系统感官事件，或在状态改变时成功触发虚拟朋友圈内容的生成。
